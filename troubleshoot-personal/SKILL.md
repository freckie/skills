# troubleshoot-personal

The user records every resolved dev issue as a troubleshooting note in their Personal Obsidian Vault. This skill covers how to reuse and write those notes.

Load alongside `obsidian-cli`, `obsidian-conventions`, and `obsidian-markdown`. Access the vault **only** through the `obsidian` CLI, never the filesystem directly.

## Location

- Notes: `20 Work/24 Troubleshooting/`
- Index: `20 Work/24 Dashboard.md`
- Template: `90 Common/99 Templates/Troubleshooting Template.md`

## Flow

1. **On a new issue** — search existing notes first; if it was solved before, reuse that note.
2. **After resolving** — if no note exists for it, write a new one.

## Finding an existing note

- Start with `20 Work/24 Dashboard.md`: match by title, `Category`, and `Keywords`.
- The dashboard table is auto-generated by the Dataview Serializer plugin from each note's frontmatter — **do not edit it by hand**.
- Notes with missing frontmatter show `-` in the dashboard and won't match by keyword. So always **also** run a full-text search as a fallback: `obsidian search:context query="<terms>" path="20 Work/24 Troubleshooting"`.

## Writing a new note

Create it from the template (`obsidian create ... template="Troubleshooting Template"`) so the frontmatter and headings are correct.

**Filename:** `YYYY-MM-DD <short description>.md` (date = resolution date). Keep within Obsidian naming limits (see `obsidian-conventions`).

**Frontmatter (required — the dashboard depends on it):**

- `CreatedAt`: `YYYY-MM-DD`
- `Status`: `해결 완료` (use `문제 발생` only if still unresolved)
- `Category`: one value. Reuse an existing one: `Linux`, `Backend`, `Container`, `CNCF 생태계`, `Cloud`, `ML`, `CICD`, `macOS`, …
- `Keywords`: list. Reuse existing tags: `K8S`, `Ceph`, `ArgoCD`, `GPU`, `Spring`, `Python`, `AWS`, `Network`, `Storage`, `Container`, … Add a new one only if nothing fits.
- `tags`: keep `troubleshooting`.

**Body:** follow the template's H2 sections — `문제 상황`, `원인`, `해결방법`, `참고` (some older notes differ; new notes use this structure). Write every section in 개조식 bullet lists.

**Linking:** link recurring concepts (tech, product) with `[[Wikilinks]]` to Topic notes under `90 Common` (see `obsidian-conventions`).
