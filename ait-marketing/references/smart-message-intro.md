# 스마트 발송

앱인토스의 스마트 발송을 활용하면 푸시/알림 메시지를 통해 신규 사용자 유입, 리텐션 향상, 서비스 이용 활성화 등의 목적을 효과적으로 달성할 수 있어요.\
메시지 유형은 광고성과 기능성으로 나뉘고, 스마트 발송은 광고성에만 해당해요.

* **광고성 메시지**: 마케팅 수신에 동의한 사용자에게 발송해요.
* **기능성 메시지**: 서비스 이용에 필요한 알림 수신에 동의한 사용자에게 발송해요.

***

## 스마트 발송이란 무엇인가요

스마트 발송은 발송 대상(세그먼트)과 발송 시점을 함께 고려해 자동으로 최적화하는 푸시/알림 발송 도구예요.\
단순 일회성 발송이 아니라, AI(Machine Learning)가 다음을 예측해 캠페인을 자동 운영해요.

* **서비스 고관여자**: 설정한 세그먼트의 pCTR(예상 클릭률)을 활용해 서비스 고관여자를 예측해요.
* **클릭 가능성이 높은 발송 시간**: 사용자의 패턴을 활용해 클릭이 더 잘 발생할 수 있는 시간대를 예측해요.

### 작동 방식

**(1) 테스트 발송**\
파트너사의 테스트가 아닌, 내부에서 AI 학습을 위해 진행하는 자동 발송이에요.

* pCTR을 기반으로 서비스 고관여자를 예측해요.
* 예측 결과를 바탕으로 최적의 사용자 및 발송 시점을 탐색해요.

**(2) 본 발송**\
테스트 결과를 반영해 캠페인을 자동 최적화하여 발송해요.

**캠페인 활성화 시간에 따른 발송 일정**

| **캠페인 활성화 시간** | **테스트 발송** | **본 발송 시작** |
| ---------------------- | --------------- | ---------------- |
| 00:00 ~ 04:59          | 당일 오전 8시   | 당일 오후 3시    |
| 05:00 ~ 11:59          | 당일 오후 3시   | 다음 날 오전 8시 |

### 푸시와 알림의 개념

* **푸시**: 앱을 열지 않은 상태에서도 받는 운영체제(OS) 알림이에요. 앱 이름과 로고가 함께 노출돼요.
* **알림**: 토스 앱 오른쪽 상단의 종 아이콘을 눌렀을 때 확인할 수 있는 앱 안의 메시지예요.

![](/assets/smart_message_intro_1.hHKE8nmJ.webp)

### 메시지 유형

메시지 내용에 따라 광고성 메시지와 기능성 메시지 중 하나를 선택해 보낼 수 있어요.

* **광고성 메시지**\
  할인, 이벤트, 신규 상품 안내처럼 마케팅 목적의 메시지예요.
* **기능성 메시지**\
  서비스 이용 과정에서 발생하는 주문, 결제, 배송, 게시글 등록 등 필수 정보를 전달하는 메시지예요.\
  구매 유도, 서비스 이용 유도, 혜택 안내, 리텐션 목적의 마케팅 요소는 기능성으로 발송할 수 없어요.

![](/assets/smart_message_intro_2.Hy56nX_7.webp)

서비스 이용 및 구매 유도와 같은 메시지가 포함되면 기능성 메시지로 발송할 수 없어요.\
메시지 문구가 고민된다면 디자인 가이드의 **[UX 라이팅 문서](https://developers-apps-in-toss.toss.im/design/ux-writing.html)** 와 아래 [메시지 작성 가이드](#_3-메시지-작성-가이드)를 참고해 주세요.

## 스마트 발송을 사용하면 어떤 점이 좋나요

* 사용자에게 꼭 필요한 메시지를 자동으로 전달할 수 있어요.
* 기능성 안내부터 마케팅 메시지까지 목적에 맞게 활용할 수 있어요.
* 사용자가 미니앱을 열지 않아도 중요한 정보를 놓치지 않게 할 수 있어요.
* 적절한 타이밍의 메시지로 서비스 이용 흐름을 자연스럽게 이어갈 수 있어요.
* 발송 결과를 바탕으로 메시지 내용을 지속적으로 개선할 수 있어요.

::: tip 참고해 주세요

* 푸시/알림을 보내려면 메시지 템플릿 문구 검수가 필요하며, 검수는 영업일 기준 2~3일이 소요돼요.
* 동일한 마케팅 목적(세그먼트, 랜딩 URL, 소재 등)으로 여러 캠페인을 운영할 수 없어요.
  * 동일한 마케팅 목적으로 여러 캠페인 생성 및 검토 요청을 할 경우, 어뷰징으로 간주되어 반려될 수 있어요.
* 푸시/알림 메시지 내용은 각각 작성하지 않아도 돼요.
  * 푸시 내용을 작성하면 알림 메시지에 자동 반영돼요.
  * 알림의 제목은 미니앱 이름으로 자동 설정돼요.
* 워크스페이스(사업자)당 10만 건까지 발송할 수 있어요.
* 기능성 메시지에 서비스 유도나 혜택 안내, 리텐션 등 광고 목적의 내용이 포함되면 광고성 메시지로 발송해야 해요.
  * 기능성 메시지를 발송할 경우 사용자에게 사전에 해당 목적으로 발송하겠다는 알림 동의가 필요해요.
  * 사용자가 알림 수신을 해제할 수 있는 기능을 제공하고, 해제 방법을 명확하게 안내하는 것을 권장해요.
  * 토스는 기능성 알림에 대해 별도의 동의 절차를 운영하고 있고, 사용자는 언제든 동의를 철회할 수 있어요.

:::

![](/assets/smart_message_intro_3.rD75sema.webp)

## 콘솔에서 설정하기

* **접속 방법**: 앱인토스 콘솔 → 워크스페이스 선택 → 미니앱 선택 → 좌측 메뉴 '스마트 발송' 선택

### 1. 광고성 캠페인

광고성 캠페인은 고객의 구매를 유도하거나 마케팅 목적으로 발송하는 메시지예요.

![](/assets/smart_message_3.yXM6Ynoc.webp)

#### 캠페인 생성

**`+생성하기`** 를 눌러 캠페인을 만들어 주세요.

* 캠페인의 목적이 드러나도록 제목을 설정하는 것을 권장해요.

#### 그룹 이름

캠페인 안에서 메시지를 관리하기 위한 그룹 이름을 설정해 주세요.

* 발송 목적이나 실험 의도가 드러나는 이름을 사용하는 것을 권장해요.

#### 타겟

광고성 메시지는 '마케팅 수신 동의'를 한 사용자에게만 발송할 수 있고, 세그먼트로만 발송할 수 있어요.\
메시지를 받을 세그먼트를 선택해 주세요.

* 세그먼트는 좌측 메뉴의 세그먼트에서 만들 수 있어요.
* 세그먼트 콘솔 가이드는 [세그먼트 가이드](/segment/intro.html)를 참고해 주세요.

#### 랜딩 URL

푸시나 알림을 눌렀을 때 이동할 URL을 입력해 주세요.

* 하나의 캠페인에서는 소재별로 다른 랜딩 URL을 설정할 수 없어요.\
  다른 URL을 사용하려면 캠페인을 새로 만들어야 해요.
* 실제로 정상 접속되는지 꼭 확인해 주세요.

#### 발송 기간

캠페인이 시작되는 시점과 종료되는 시점을 설정해요.

* 소재 검토 기간(최대 3영업일)과 최적화 테스트 발송 기간(최대 7일)을 고려해 넉넉하게 설정하는 것을 권장해요.
* 캠페인 활성화 기간 중에 일자를 수정하면 최적화 발송이 처음부터 다시 진행되니 유의해 주세요.

#### 메시지

광고성 메시지 템플릿을 선택하고 서비스에 맞게 수정해 주세요.

* 자세한 작성 방법은 [소재 작성 가이드](#_3-메시지-작성-가이드)를 참고해 주세요.

***

### 2. 기능성 캠페인

기능성 캠페인은 서비스 이용에 직접적으로 필요한 정보를 제공하기 위한 메시지예요.\
수신자가 요청한 정보이거나 서비스 이행을 위한 필수 메시지이어야 해요.\
서비스 사용 유도나 혜택 안내, 리텐션 등 광고 목적의 내용이 포함되면 광고성 캠페인으로 발송해야 해요.

:::details 허용되는 예시

* 결제 완료, 환불, 포인트 적립 등 거래 관련 메시지
* 배송 시작, 배송 완료, 예약 확정 등 서비스 진행 상황 메시지
* 정기 결제 예정, 서비스 만료 예정 등 이용 기간/상태 알림
  :::

:::details 허용되지 않는 예시

* "이런 상품은 어떠세요?" / "오늘만 할인 중이에요." → 판촉/광고 목적
* "이전 구매 고객을 위한 추천 상품" → 부가적 마케팅 정보
* "이벤트 참여하고 혜택 받아가세요." → 이벤트 홍보성 메시지
  :::

#### 2-1. 알림 동의문

기능성 메시지를 보내려면 알림 동의문이 필요할 수 있어요.\
알림 동의문은 사용자가 서비스 이용 과정에서 알림 수신에 동의할 수 있도록 제공하는 안내 문구예요.\
특정 시점에 알림을 받겠다고 사용자에게 동의를 받는 경우에는 반드시 알림 동의문을 사용해야 해요.

알림 동의문이 필요한 경우예요.

* 재입고 알림 신청, 이벤트 시작 알림 신청, 가격 변동 알림 신청, 예약 오픈 알림 신청

반면 사용자가 서비스 이용에 꼭 알아야 하는 내용은 알림 동의문 없이도 보낼 수 있어요.

* 결제 완료, 배송 시작, 배송 완료, 환불 완료, 정보 변경 안내, 약관 변경 안내

알림 동의문이 필요한 메시지인데 동의문을 등록하지 않으면 기능성 메시지 템플릿을 저장할 수 없어요.

**① 알림 동의문 등록하기**

* 앱인토스 콘솔 → 워크스페이스 선택 → 미니앱 선택 → 좌측 메뉴 '스마트 발송' → **알림 동의문** 탭 선택
* **`등록하기`** 를 눌러서 알림 동의문을 등록할 수 있어요.

![](/assets/notification_agreement_1.CL4Gm5SX.webp)

**② 알림 동의문 설정하기**

![](/assets/notification_agreement_2.BXIKE1aM.webp)

* **알림 동의문 이름** — 미니앱 서비스 내에서 어느 상황에서 해당 동의문이 활용되는지 알 수 있게 작성해 주세요.
* **알림 발송 시점** — 동의할 경우 언제 발송하는지 작성해 주세요.
  * 예: 찜한 상품 가격이 인하되었을 때, 매일 날씨 안내를 받기로 했을 때, 서비스 내 정지된 기능이 다시 활성화되었을 때 등
* **알림 동의문에는 다음 내용이 포함돼요.**
  * 서비스명과 서비스 설명 (미니앱 정보에서 자동 반영)
  * 알림 발송 시점 (템플릿 생성 시 입력한 트리거 조건)
  * 알림 동의 철회 경로
  * 고객센터 연락처 (미니앱 정보에서 자동 반영)
* **알림 발송 방법**
  * 특정 조건 충족: 어떤 조건이 충족되면 알림을 발송하는지 입력해 주세요. (예: 찜한 상품 가격 인하, 정지된 기능 재활성화 등)
  * 정해진 요일과 시간: 발송 주기와 유형을 입력해 주세요. (예: 매일 날씨 안내, 출석 알림 등)

![](/assets/notification_agreement_3.DuwnKTHJ.webp)

미니앱에서 사용자에게 알림 수신 동의 UI를 표시하려면 [알림 동의문 요청하기](/bedrock/reference/framework/인터렉션/requestNotificationAgreement.html) 문서를 참고해 주세요.

#### 2-2. 기능성 메시지 발송하기

**① 직접 API로 발송하기**

파트너사 서버에서 원하는 시점에 발송할 수 있어요.\
발송 API 상세 스펙은 [기능성 메시지 발송하기](/smart-message/develop.html) 문서에서 확인할 수 있어요.

![](/assets/notification_agreement_4.DM3zYVAi.webp)

* **캠페인 제목** — 어떤 목적으로 발송되는 캠페인인지 알기 쉽게 제목을 설정해 주세요.
* **제목** — 7자 이내 (띄어쓰기 포함), 명사형으로 작성해 주세요.
  * \~하기 형태로 작성하면 사용자 행동 유도 목적으로 오인될 수 있어요.
* **내용** — 25자 이내 (띄어쓰기 포함), ~요. 체로 작성해 주세요.
  * 변수는 2글자로 간주돼요.
* **이동 URL** — 푸시 또는 알림을 눌렀을 때 이동되는 페이지를 설정해 주세요. 접속이 되는지 꼭 확인해 주세요.
* **알림 동의문** — 문구 검토 시 알림 동의문이 필요하다고 판단되어 반려되는 경우, 반드시 알림 동의문 구성 후 재검토 요청해 주세요.
  * [알림 동의문 구성하는 방법 보러 가기](#_2-1-알림-동의문)

![](/assets/notification_agreement_5.DvVqiTen.webp)

![](/assets/notification_agreement_6.CKjqXQe1.webp)

**② 토스에게 발송 요청하기**

서버 없이 일회성 발송 및 정기 발송을 선택해서 발송할 수 있어요.

![](/assets/notification_agreement_7.C5MZgQ1_.webp)

* **캠페인 제목** — 어떤 목적으로 발송되는 캠페인인지 알기 쉽게 제목을 설정해 주세요.
* **제목** — 7자 이내 (띄어쓰기 포함), 명사형으로 작성해 주세요.
* **내용** — 25자 이내 (띄어쓰기 포함), ~요. 체로 작성해 주세요.
  * 변수는 2글자로 간주돼요.
* **이동 URL** — 푸시 또는 알림을 눌렀을 때 이동되는 페이지를 설정해 주세요. 접속이 되는지 꼭 확인해 주세요.
* **알림 동의문** — 토스에게 발송 요청하기를 통해 발송하는 경우 알림 동의문이 반드시 필요해요.

#### 2-3. 발송하기

문구 소재 검토 승인 후 발송을 진행할 수 있어요.

* **직접 API로 발송하는 경우** — 파트너사 서버에서 원하는 시점에 발송할 수 있어요.
* **토스에 발송 요청하는 경우** — 일회성 발송 및 정기 발송을 선택해서 발송할 수 있어요.

### 3. 메시지 작성 가이드

토스를 쓰는 모든 사용자에게 내 앱을 처음 소개한다는 마음으로 작성해 주세요.\
사용자는 이 푸시를 '토스'라는 금융앱의 맥락 안에서 받아들여요.\
메시지가 토스답게 보이고, 자연스럽게 이해될 수 있도록 써 주세요.

::: tip 꼭 지켜 주세요

* 맞춤법 및 띄어쓰기를 꼭 확인해 주세요.
* **메시지 내용은** 문장형(~요.)으로 작성해 주시고, 문장 마지막에는 꼭 마침표를 사용해 주세요.
* 토스 + 서비스명으로 작성하는 방식은 쓸 수 없어요.
* 제목은 ~하기, 명사형으로 작성해 주세요. (기능성은 명사형으로 작성해 주세요.)

:::

#### 기본 규칙

#### 타이틀은 공백 포함 `최대 7자`, 본문은 `25자`까지 쓸 수 있어요

#### 제목 쓰는 법

**1) 제목과 내용이 자연스럽게 연결돼야 해요.**

:::tip ✅ 좋은 예시

* **퍼즐 도전** / 토스에서 퍼즐 맞추고 포인트 받아 가세요.
* **출석 선물** / 블록 게임 3일 출석하면 20원을 드려요.
* **금전운 도착** / 토스에서 무료로 운세를 확인해보세요.
  :::

:::danger ❌ 나쁜 예시

* **떡볶이로?** / 토스에서 MBTI 심리테스트 진단하세요.
  * → 제목은 음식인데 내용은 MBTI라서 연결이 안 돼요.
* **마이너스 통장** / 공모주 청약에 쓸 수 있어요.
  * → 통장을 만들라는 건지, 청약 소식을 알리는 건지 모호해요.
    :::

**2) 제목은 그 자체로 완성된 단어나 문장이어야 해요.**

:::danger ❌ 문장 카드로
토스에서 39종 카드 골라보세요.
:::

:::tip ✅ 문장 카드
토스에서 명언 고르고 원하는 디자인 선택해요.
:::

:::danger ❌ 숏폼보다
토스에서 39종 카드 골라보세요.
:::

:::tip ✅ 블록 게임
시간가는 줄 모르는 게임에 도전해보세요.
:::

#### 반드시 포함할 내용

**1) 비게임 서비스는 '토스에서'를 넣어요**

'토스에서' 또는 '토스'가 없으면 토스 앱이 아닌 외부 페이지로 이동한다고 생각해요.

:::danger ❌ 재회할 확률
사주로 그 사람과 다시 만날 확률을 알아봐요.
:::

:::tip ✅ 재회할 확률
토스에서 사주보고 재회 확률을 알아봐요.
:::

**2) 게임 서비스는 '게임'이라고 밝혀요**

사용자가 클릭하기 전에 게임인지 알아야 해요. 서비스명이나 게임 내 용어만 쓰면 모를 수 있어요.

:::danger ❌ 경쟁하는 퍼즐
퍼즐 맞추고 내 집도 꾸며요.
:::

:::tip ✅ 퍼즐 게임
퍼즐 맞추고 내 집도 꾸며요.
:::

#### 작성 시 사용 불가한 표현

**1) 일부만 이해할 수 있는 은어, 밈, 유행어**

:::danger ❌ 엠블렘 등장
오늘 새로 나온 장비 콘텐츠예요.
:::

:::tip ✅ 새 아이템
오늘 새로 나온 게임 아이템이 있어요.
:::
→ 게임 내부 용어

:::danger ❌ 애슬레저 맛집
젝시믹스에서 토스로 결제하면 4천원 할인 받아요.
:::

:::tip ✅ 젝시믹스 할인
토스페이로 결제하면 최대 4천원 할인 받아요.
:::
→ 밈, 유행어

**2) 특정인의 이름**

:::danger ❌ 내 이상형
장원영인가요, 안유진인가요?
:::

:::tip ✅ 이상형 월드컵
토스에서 이상형에 가까운 아이돌을 골라봐요.
:::

**3) 과장된 광고성 표현**

* 광고성 표현: 지금 바로!, 초특가!, 역대급, 긴급!, 대박
* 불안 조성: 놓치면 후회해요, 지금 안 하면 못 해요
* 과도한 기호: 느낌표, 이모지

**4) 자극적인 소재**

① 정치, 범죄, 사망, 민감한 소재는 피해요.

:::danger ❌

* **`불륜` 지수** / 토스에서 재미로 테스트해봐요.
* **오늘의 팁 /** `임신 중 남편이 사망하면` 상속순위는 어떻게 될까요?
  :::

② 불안감을 조성하지 않아요.

놓치면 후회한다는 뉘앙스나, 사용자를 과도하게 불안하게 하는 단어는 쓰지 않아요. 상황만 차분히 전달해 주세요.

:::danger ❌

* 아직도 **안써보셨나요?** / **예산을 항상 초과해도 괜찮으신가요?**
* 유효기간이 **곧 끝나요** / 정말 혜택을 **놓치시겠어요?**
  :::

**5) 띄어쓰기 생략**

글자수를 맞추기 위해 띄어쓰기를 생략하지 않아요.

:::danger ❌ 내타자정확도는
눈 감고 문장 치기 도전해보세요.
:::

:::tip ✅ 내 타자 실력
토스에서 눈감고 문장치는 테스트해보세요.
:::

:::danger ❌ 위로가필요할때
나로가 처방해주는 책과 음악으로 힐링해요.
:::

:::tip ✅ 음악 추천
토스에서 위로되는 음악을 처방해 드릴게요.
:::

**6) 어색한 표현**

문법적으로 틀리진 않지만 자연스럽지 않은 표현이에요. 소리 내어 읽어보면 금방 알 수 있어요.

:::danger ❌ 나만의 소설
다른 장르도 선택해 보세요. 토스 무료예요.
:::
→ 조사가 빠져서 어색해요. '토스' 서비스를 이용하는 게 무료라고 읽혀요.

:::tip ✅ 나만의 소설
토스에서 무료로 만들 수 있어요.
:::
