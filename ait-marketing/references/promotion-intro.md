# 프로모션

프로모션으로 가벼운 클릭이 자연스럽게 전환까지 이어지도록 기획해 보세요.\
재방문을 유도하고 자연스러운 바이럴까지 퍼널 곳곳에서 성과를 낼 수 있어요.

***

## 프로모션이란 무엇인가요

앱인토스의 프로모션은 사용자의 특정 행동을 기준으로 토스 포인트를 지급하는 이벤트예요.\
예를 들어 가입하기, 첫 이용 완료 같은 행동을 기준으로 혜택을 제공할 수 있어요.

프로모션은 비즈 월렛에 충전한 예산으로 운영해요.\
혜택 탭에 노출할지 여부도 콘솔에서 직접 설정할 수 있어요.

![](/assets/promotion-intro-1.BX6JhrS9.webp)

::: tip 꼭 확인해 주세요

프로모션 기능은 별도의 서버 없이도 사용할 수 있어요.

비게임 미니앱의 경우, 서버를 통해 프로모션 포인트를 지급하는 방식(Server-to-Server)을 추가로 제공해요.\
요청 위변조 방지 등 무결성이 중요한 경우에는 서버를 통한 지급 방식을 이용할 수 있어요.

서버를 통해 프로모션을 지급하는 경우, 토스 로그인을 구현해야 사용할 수 있어요.\
[토스 로그인 가이드](/login/intro.html)를 참고해 주세요.

:::

***

## 포인트가 지급되면 어떻게 보이나요

프로모션을 통해 토스 포인트가 지급되면, 유저 화면 상단에 토스트(toast) 알림이 표시돼요.

* {{미니앱 이름}}에서 {{금액}}원을 지급했어요 형식으로 일시적 노출되며 자동으로 사라져요.
* 토스 앱 알림 메뉴(홈화면의 종 버튼)에서도 확인할 수 있어요.
  * 단, 유저의 알림 설정 여부에 따라 알림 메뉴에서 확인이 되지 않을 수 있어요.
* 혜택 탭 내의 토스포인트 메뉴에서 적립 내역도 확인할 수 있어요.
  * 토스앱 → 혜택 → 토스포인트 선택 → 포인트 금액 선택 → 미니앱 이름으로 적립 내역 확인 가능

![](/assets/promotion-intro-2.kxU0_Nuw.webp)

## 프로모션을 사용하면 어떤 점이 좋나요

* 비즈 월렛 선충전과 예산, 종료일 설정으로 초과 집행 걱정 없이 운영할 수 있어요.
* 토스 혜택 탭에 노출되면 많은 트래픽과 함께 더 많은 사용자가 유입될 수 있어요.
* 가입하기, 첫 플레이 진행하기처럼 사용자의 행동을 간단하게 유도할 수 있어요.
* 일간이나 주간 미션으로 사용자가 다시 방문할 이유를 만들어 리텐션을 높일 수 있어요.
* 포인트 지급과 결과 조회를 API로 처리할 수 있어요.
* 테스트용 코드인 `TEST_{promotionCode}`를 사용하면 실제 포인트 차감 없이 검증할 수 있어요.

::: tip 참고해 주세요

* 앱인토스의 여러 프로모션을 모아 보여주는 '새로운 서비스 써보고' 탭은 내부 노출 로직에 따라 일부 사용자에게만 보여질 수 있어요.
* 앱인토스의 여러 프로모션을 모아 보여주는 '새로운 서비스 써보고' 탭은 내부 노출 로직에 따라 일부 사용자에게만 보일 수 있어요.
* 사행성 콘텐츠, 자사 앱 설치 유도, 법령 위반 가능성이 있는 콘텐츠로는 프로모션을 진행할 수 없어요.
* 1인 1회 제한, 일일 제한, 쿨다운, 중복 지급 방지 로직은 반드시 적용해 주세요.
* 지급이 지연될 경우 PENDING 상태 안내와 결과를 확인할 수 있는 경로를 제공해 주세요.
* 프로모션을 시작하려면 사전 검수가 필요하고, 검수에는 영업일 기준으로 약 2~3일이 걸려요.

:::

## 콘솔에서 설정하기

* **접속 방법**: 앱인토스 콘솔 → 대표 관리자 로그인 → 워크스페이스 선택 → 미니앱 선택 → 좌측 메뉴 '프로모션' 선택

### 진행 전 준비사항

프로모션을 시작하려면 다음 항목을 먼저 준비해 주세요.\
이미 준비가 완료됐으면 바로 [5번(프로모션 등록)](#_5-프로모션-등록)으로 이동해도 돼요.

![](/assets/promotion_console_0.vIhW7DfH.webp)

* 사업자 정보 등록 및 정산 정보 검토 (검토에 영업일 기준 1~2일, 정산 검토 2~3일 소요)
* 대표 관리자의 프로모션 기능 약관 동의 및 비즈 월렛 약관 동의
* 프로모션 예산을 비즈 월렛에 충전 (최소 30만 원, 최대 3,000만 원)

### 1. 사업자 정보 등록

프로모션은 사업자(개인 또는 법인) 계정에서만 사용할 수 있어요.\
사업자 등록이 되어 있지 않으면 콘솔에서 '먼저 사업자 정보를 등록해주세요' 화면이 보여요.\
'등록하기'를 눌러 사업자 정보를 등록해 주세요.

* 사업자 정보 검토는 **영업일 기준 약 1~2일** 소요될 수 있어요.
* 자세한 절차는 [가이드](/prepare/register-business.md)를 확인해 주세요.

![](/assets/promotion_console_1.B2tjyX9p.webp)

### 2. 정산 정보 등록

사업자 등록이 완료되면 '다음으로 정산 정보를 등록해주세요' 화면으로 변경돼요.\
'등록하기' 버튼을 눌러 파트너 정보 메뉴에서 정산 정보를 등록하고, 검토 요청을 해 주세요.

* 정산 정보 검토는 **영업일 기준 평균 2~3일** 소요돼요.
* 정산 정보 등록 전에는 검토 요청이 불가하니 사업자 등록 상태를 먼저 확인해 주세요.

::: tip 정산 등록 시 주의사항

* 예금주명은 통장 사본과 **정확히 일치**하게 입력해 주세요.
* 업태·업종은 숫자 코드가 아니라 **텍스트**로 입력해 검색해 주세요.\
  예) '소프트웨어 개발업' → 소프트웨어로 검색

:::

### 3. 비즈 월렛 충전

정산 정보 검토가 완료되면 워크스페이스의 '비즈 월렛' 메뉴에서 예산을 충전할 수 있어요.

* 충전 가능 금액: **최소 300,000원 ~ 최대 30,000,000원**
* 현재 결제 수단: 신용카드(인증/비인증 결제)
  * 인증결제: 카드사별 결제 제한 없음
  * 비인증결제: 하나카드 1일 100만 원 제한, 기타 카드사 제한 없음\
    (비인증 결제는 카드 번호 직접 입력 방식이에요.)

충전이 완료되면 상태가 충전으로 바뀌고 충전액이 반영돼요.

![](/assets/promotion_console_4.dYjJ42WM.webp)

### 4. 프로모션 허용/불가 유형 확인

#### 허용되는 유형 (모두 1인당 최대 5,000포인트 이내)

* 회원가입/접속 보상 (예: 신규 가입 시 2,000포인트 지급)
* 거래/구매 유도형 (조건과 고지가 명확해야 함, 예: 5,000원 이상 결제 시 500포인트 지급)
  * 단, 한 번 지급된 토스 포인트는 회수할 수 없으니 프로모션 조건과 지급 기준을 충분히 검토한 뒤 신중하게 운영해 주세요.
* 이벤트 참여형 (설문·퀴즈·간단 미션, 단 과도한 시간·노동 요구 불가)
* 친구 초대형 (초대한 사람과 초대받은 사람 모두 보상, 어뷰징 방지 로직 필수)
* 비게임 앱의 확률형/랜덤 보상
  * 프로모션 기간은 1주일 이내로만 운영할 수 있어요.
  * 새로운 사용자 유입을 위한 홍보 목적으로만 가능해요.
  * 비게임 앱으로 등록되었더라도 게임물로 분류된 경우 확률형 프로모션 집행이 불가해요.

#### 허용되지 않는 유형

* 게임 앱의 확률형/랜덤 보상 (룰렛·뽑기 등)
* 게임 결과 기반 보상 (점수·승패·등수 기반)
* 재화 환전형 리워드 (기프티콘·상품권 → 토스 포인트 전환)
* 현금 보장형·유사 수신 행위 (투자성·사행성 성격)
* 1인당 5,000포인트 초과 지급 (추첨형은 별도 검토)

### 5. 프로모션 등록

대표 관리자가 약관 동의와 비즈 월렛 충전을 완료하면 프로모션 등록이 가능해요.

등록하기 전에 [프로모션 허용/불가 유형](#_4-프로모션-허용-불가-유형-확인)을 확인해 주세요.

* 경로: 앱인토스 콘솔 → 대표 관리자 로그인 → 워크스페이스 선택 → 미니앱 선택 → 좌측 메뉴 프로모션 → +등록하기

![](/assets/promotion_console_7.BmLa4nGV.webp)

#### 혜택 탭 노출하는 경우

![](/assets/promotion_console_8.B5DxXkqJ.webp)

#### 혜택 탭 미노출하는 경우

![](/assets/promotion_console_9.B4BjlV3o.webp)

1. **프로모션 이름** — 사용자가 어떤 조건을 만족하면 포인트를 받는지 구체적으로 입력해 주세요.\
   예) 서비스 로그인 시 10포인트 지급, 튜토리얼 진행 완료 시 20포인트 지급

2. **프로모션 종료일** — 희망 종료일을 설정해 주세요. 예산 소진 시 예정일보다 먼저 종료될 수 있어요.

3. **혜택 탭 노출 여부**
   * 노출: 토스 앱 → 혜택 → 새로운 서비스 써보고 메뉴에 함께 노출돼요.
   * 미노출: 미니앱을 통해 직접 접속하는 유저에게만 프로모션을 제공해요.
   * 주의: 미노출로 등록 후 나중에 노출하려면 **새 프로모션 등록**이 필요해요. (수정으로는 변경 불가)

4. **미션 이름** (혜택 탭 노출 시 필수) — 미션 이름은 ~하기로 끝나야 해요.\
   예) 튜토리얼 하기, 로그인 하기, 리뷰 남기기

5. **지급 방식** (혜택 탭 노출 시 필수)
   * 고정 금액: 정해진 금액 지급
   * 최대 금액: 1인당 최대 지급 가능 금액(랜덤 지급 범위 내)\
     예) 최대 100원 → 1회 참여 시 0~100원 중 지급

6. **이동 URL** (혜택 탭 노출 시 필수)
   * 형식: `intoss://{{appName}}/ScreenName`
   * 혜택 탭에서 이 URL로 진입하게 돼요.

7. **예산 정보**
   * 예산은 비즈 월렛 잔액보다 크게 설정할 수 없어요.
   * 진행 중 예산 증액이 필요하면 비즈 월렛 충전 후 프로모션 수정으로 증액 가능해요.

프로모션 정보를 입력한 뒤 '검토 요청하기'를 눌러 주세요. 검토는 영업일 기준 2~3일 소요돼요.

### 6. 테스트 진행

검토가 완료되면 '테스트하기'로 테스트를 진행해 주세요. 테스트는 실제 포인트를 차감하지 않아요.

* 테스트 프로모션 코드는 실제 코드 앞에 `TEST_`가 붙어요.
* 테스트 시 포인트는 차감되지 않고 실제 지급도 이뤄지지 않아요.
* 포인트 지급 API 호출 시 `resultType`이 `SUCCESS`로 응답되는지 확인해 주세요.
* 검토 완료 후 최소 1번의 포인트 지급 API 테스트가 완료되어야 프로모션을 시작할 수 있어요.

![](/assets/promotion_console_11.7gcmx3Dw.webp)

### 7. 프로모션 시작 및 운영

'시작하기' 버튼을 눌러 프로모션을 시작해 보세요.

* 예산 열은 `실제 소진 금액 / 총 예산` 형태로 표시돼 예산 소진률을 확인할 수 있어요.
* 프로모션 수정은 예산 증액에 주로 사용해요.
* 프로모션 종료 시 남은 예산은 비즈 월렛으로 환급돼요.
* 예산 소진 시 혜택 탭에는 더 이상 노출되지 않으니 예산 관리를 주기적으로 해 주세요.

![](/assets/promotion_console_12.DgzBD0Kd.webp)

캠페인 이름을 누르면 프로모션 상세 정보를 확인하고 수정, 종료, 일시정지를 할 수 있어요.

* **수정** — 주로 예산을 증액할 때 사용해요.\
  예산 증액 방법: 캠페인 상세 → 프로모션 수정 버튼 → 증액할 금액 입력 → 저장\
  예산 외 설정을 변경하려면 내부 정책상 제한이 있을 수 있으니 검토 가이드라인을 먼저 확인해 주세요.
* **종료** — 프로모션이 종료되면 남은 예산은 자동으로 비즈 월렛으로 환급돼요.\
  환급 금액은 종료 후 즉시 반영되는 경우가 많지만, 시스템 처리로 인해 약간의 지연이 발생할 수 있어요.

::: tip 간단한 운영 팁

* 예산을 자주 모니터링해서 소진으로 인한 자동 종료를 방지해 주세요.
* 혜택 탭 노출 여부는 수정으로 변경할 수 없으니 노출 설정은 등록 전 최종 확인해 주세요.

:::

![](/assets/promotion_console_13.CNlCBdZA.webp)

### 8. 성과 대시보드 확인

프로모션을 시작하면 해당 프로모션의 성과를 대시보드에서 확인할 수 있어요.\
이 대시보드는 프로모션이 얼마나 효과적이었는지 한눈에 보여 주고, 이후 캠페인 전략을 개선하는 데 도움을 줘요.

* 접속 방법: 앱인토스 콘솔 → 워크스페이스 선택 → 미니앱 선택 → 프로모션 → 성과 탭

#### 기간 설정 및 조회 조건

성과 데이터는 다양한 기준으로 조회할 수 있어요. 분석 목적에 맞게 조건을 바꿔가며 살펴보면 좋아요.

* **일별, 주별, 월별** 단위로 데이터를 조회할 수 있어요.
* **운영체제(OS)별**로 나눠서 성과를 확인할 수 있어요.
* 현재 미니앱에서 운영 중인 전체 프로모션과 개별 프로모션을 구분해서 조회할 수 있어요.

**혜택 탭에 노출하지 않은 프로모션은 성과 대시보드에 표시되지 않아요.**

#### 최종 전환율

프로모션 성과 대시보드에서는 유저가 프로모션에 참여하는 흐름을 단계별로 확인할 수 있어요.\
이 흐름을 보면 **최종 전환율**을 알 수 있고, 어느 단계에서 유저가 많이 이탈하는지도 파악할 수 있어요.

![](/assets/promotion_dashboard_1.C9zZ6-jg.webp)

1. **혜택탭 진입** — 토스 앱의 혜택탭에서 해당 프로모션이 노출되고, 이를 확인한 유저 수예요.
2. **프로모션 클릭** — 혜택탭에서 프로모션을 실제로 클릭해 상세 화면으로 진입한 유저 수예요.\
   혜택탭 진입 대비 클릭 수를 보면, 프로모션 제목이나 썸네일이 유저의 관심을 얼마나 끌었는지 알 수 있어요.
3. **포인트 획득** — 프로모션 조건을 충족하고 토스 포인트를 실제로 지급받은 유저 수예요.\
   조건이 어렵거나 과정이 복잡하면 이 단계에서 이탈이 크게 발생할 수 있어요.

#### 유입 유저

프로모션으로 유입된 유저의 성격을 기준으로 성과를 나눠서 확인할 수 있어요.

![](/assets/promotion_dashboard_2.CtZQXSom.webp)

* **신규 유저** — 프로모션을 계기로 처음 미니앱에 유입된 유저예요.
* **기존 유저** — 과거에 미니앱 서비스를 이용한 이력이 있는 유저예요.

#### 기여 매출

프로모션이 실제 매출에 얼마나 기여했는지도 확인할 수 있어요.\
다만, 기여 매출은 워크스페이스(사업자 번호) 단위로만 조회할 수 있고, 미니앱별로는 조회할 수 없어요.

![](/assets/promotion_dashboard_3.40MqMppB.webp)

* **총 기여 매출** — 프로모션을 통해 유입된 유저들이 발생시킨 전체 매출 금액이에요.
* **결제자 기준 1인당 평균 매출** — 프로모션으로 유입된 유저 중 **실제로 결제한 유저만**을 기준으로 계산한 평균 매출이에요.
* **전체 유저 기준 1인당 평균 매출** — 결제 여부와 관계없이 **프로모션으로 유입된 전체 유저**를 기준으로 계산한 평균 매출이에요.

### 사용자 안내 필수 항목

프로모션을 안내할 때 반드시 다음 정보를 고지해 주세요.

* 지급 시점 (예: 즉시 지급, 익일 18시 지급 등)
* 지급 조건 (예: 최초 결제, 5,000원 이상 결제 등)
* 지급 제한 (탈퇴·환불·부정 참여 시 지급 불가)
* "본 프로모션은 사전 고지 없이 중단될 수 있습니다" 문구 포함
* 랜덤 지급 불가 (고정 지급만 허용)

### 정산 및 운영 원칙

* 프로모션 예산은 비즈 월렛에서 충전 후 설정하고, 예산 소진 시 프로모션이 종료돼요.
* 프로모션 중단 시 남은 예산은 비즈 월렛으로 반환돼요 (잔액이 있을 경우).
* 중복 참여·어뷰징 방지 로직 적용은 필수예요.
* 클레임 발생 시 참여 기록 확인 후 파트너사에서 처리해야 해요.

#### 프로모션 사전 점검 체크리스트

아래 항목을 모두 충족해야 프로모션 집행이 가능해요.

□ 제공되는 포인트가 **1인당 5천 포인트 이하**인가요?\
□ 지급 조건, 지급 시점, 지급 제한 사항 등을 명확히 고지했나요?\
□ 참여 방식이 단순하고 과도한 시간 또는 노동 요구는 없나요?\
□ (게임의 경우) 룰렛·뽑기 등 확률형 요소와 결합되지 않았나요?\
□ 제공되는 포인트가 게임의 결과 또는 등수 기반으로 산정되는 것은 아닌가요?\
□ 유저가 보유한 재화를 토스포인트로 교환/전환해 주는 형태의 프로모션은 아닌가요?\
□ 진행되는 프로모션의 조기 종료 혹은 중단 가능성을 사전 고지했나요?\
□ 중복 참여 방지 로직을 적용했나요?\
□ 사행성 또는 과장된 프로모션은 아닌가요?
